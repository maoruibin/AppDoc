# 云同步

inBox 是一个本地笔记软件，数据默认存储在手机本地。为了保证数据安全并实现多设备同步，inBox 支持将笔记数据同步到个人云端。

## 支持的云服务

| 云服务 | 适用场景 | 配置教程 |
|--------|----------|----------|
| **WebDAV** | 个人云盘同步，支持坚果云、Nextcloud 等 | [配置 WebDAV](./lesson-webdav.md) |
| **S3 兼容存储** | 对象存储服务，传输速度快，适合大量数据 | [配置 S3](./lesson-web-s3.md) |
| **Memos** | 开源自托管笔记服务，支持双向同步 | [配置 Memos](./memos.md) |

在 **数据中心** 页面中选择并配置你使用的云服务即可。

## 同步模式

inBox 支持两种同步模式：

- **自动同步（推荐）**：App 启动时自动同步，数据变更时自动同步
- **手动同步**：仅在手动触发时同步

> 可在数据中心页面中切换同步模式。

## 同步机制

inBox 采用**原子化同步**方式：每条笔记独立一个文件（`notes/note-{id}.json`），单条笔记同步失败不影响其他笔记。

同步流程：
1. 笔记创建或编辑后，自动标记为待同步
2. 根据同步策略，将待同步的笔记上传到云端
3. 其他设备拉取云端数据，实现多端同步

## 上传笔记到云端

如果需要将本地所有笔记批量上传到云端（例如首次配置云服务、切换云服务、或同步异常后修复），可以使用 **上传笔记到云端** 功能。

操作路径：数据中心 → 上传笔记到云端

页面会自动扫描待上传的笔记数量，点击「开始上传」即可。上传过程中可退出页面，通过通知栏查看进度。

## 重置同步状态

当出现同步异常时，可以通过 **重置同步状态** 来修复。

操作路径：数据中心 → 重置同步状态

> **使用方法**：先点击「重置同步状态」，将所有笔记的同步状态清除，然后点击「上传笔记到云端」，将本地笔记重新批量同步到云端。

适用于以下场景：
- 云端数据损坏，需要重新上传
- 切换了云服务提供商
- 同步异常，需要强制重置

## 常见问题

### 如何把笔记数据转移到另一个手机？
使用本地备份还原功能，详情查看 [如何备份还原图谱数据](./data_recovery.md)

### 图片、视频、录音会同步吗？
这些文件存储在 inBox 的本地目录（`/Android/data/name.gudong.think/files/Documents/inBox/`）。目前云同步的是笔记数据，媒体文件可通过本地备份方式转移。

### 备份的数据包含哪些内容？
- 笔记数据（标题、内容、创建时间等）

> 标签信息存储在笔记内容中（如 `#标签/子标签`），同步后其他设备会自动重建标签关联。

### 加密是什么意思？
备份文件默认经过 AES 加密，只有 inBox 可以读取。关闭加密后，备份数据为明文 JSON 格式。
